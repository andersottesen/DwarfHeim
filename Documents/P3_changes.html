The page design


Media drop-down menu

In the mock-up of the web application we planned to have two separate pages for the media, one for images and one for videos. In the end we placed both galleries on one page. We only had three videos, and figured out that a whole page for both galleries was unnecessary. Therefore is the drop-down menu under "media" in the navigation bar removed.

The Crew header image

The original plan was to use the same header-image for all the pages, but we decided it was cool with a different header-image on “The Crew” page.

The Crew information boxes

In the mock-up of "the crew"-page, the boxes of content were placed much closer together than what we ended up with, because we decided it looked better with more space in between.

Navbar-color

In the original plans the navigation bar turns orange when hovered. In practise this did not look as good as we wanted, so we decided to just use a different transparency.

JavaScript

Since we decided not to implement two of our five planned JavaScripts, we had to make a new one in order to meet the requirements. We decided to create hp_randomizer.js which randomizes the images that are shown on the "home"-page.

When we started to implement the web application we saw that we needed to reallocate the different pages and scripts. We did most of the programming during team-meetings, to make sure that we all were on the same track. Some of the team-members needed a bit of guidance from the more experienced team members, and we found out that pair-programming, and working together was most successful. Also, one team member was not able to attend a lot of the meetings, so it was easier to give him tasks that did not require a lot of input from the rest of the team (such as CSS).

The original deadlines we had set for our self where easy to meet, therefore we have not included these in this document. The implementation of the web application took less time than expected, and we completed the implementation well within the 7th of November. The reason for this was probably good planning and a motivated team.

Changes done after testing

All heading-fonts and the color of the font were changed to match the navigation bar. The landing-page was made a bit smaller, to fit the screen, so that the user do not need to scroll. The logo was made into a link as well.

Suggested changes that arose during the user testing:

  • Improved and updated background with more decoration on the left and the right side of the page- content. One of the test users commented that the pages' background was a bit boring without any symbols or decorations except for the footer. We imagine an improvement that includes more mountains and dwarfs like in the footer.
  • There is a lack of information on the page. That is something we need to request from the client.
